
                                                                                                      6 — Set up cross-cluster Kubernetes auth

                    This is the customer-facing setup step: configure a Vault auth method in the Vault cluster that validates service account JWTs against the VSO cluster's API server.

                    What that command does

                       •  Reads the VSO cluster API server CA from kubeconfig.
                       •  Mints a reviewer JWT for vso-demo/vault-token-reviewer in the VSO cluster.
                       •  Enables/refreshes auth/kubernetes-vso in Vault without touching same-cluster auth/kubernetes.
                       •  Writes auth/kubernetes-vso/config with the VSO API endpoint.
                       •  Writes role vso-demo, bound only to service account vso-demo/vso-demo.
